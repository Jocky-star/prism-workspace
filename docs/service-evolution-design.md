# 个人AI助手服务演进设计

> 从 Day 1 到 Day 365，每个阶段怎么服务用户

---

## 核心原则

1. **每个阶段都要有用** — 不能说"等数据多了就好了"，Day 1 就要让用户觉得值
2. **从有用到懂你** — 先做工具人，再做参谋，最后做知己
3. **观察先于建议** — 不要急着给建议，先证明"我注意到你了"
4. **建议必须可执行** — "多运动"是废话，"今晚 20:30 去亚C，周二你到家最早"是建议
5. **反馈驱动迭代** — 用户没采纳的建议，降低同类权重

---

## Phase 0：冷启动（Day 1-3，无数据）

### 目标
让用户感到"这个东西立刻有用"，同时自然地收集基础信息。

### 策略：Zero-data Utilities
不需要任何用户数据就能提供的价值：

| 功能 | 实现 | 价值感 |
|------|------|--------|
| 天气播报 | 问一次城市，之后自动 | ⭐⭐⭐ 每天都用 |
| 新闻摘要 | AI 筛选 + 一句话评论 | ⭐⭐ 信息密度 |
| 快速提醒 | "提醒我明天10点开会" | ⭐⭐⭐ 即时有用 |
| 翻译/查询 | 随问随答 | ⭐⭐ 工具属性 |
| 倒计时/纪念日 | 主动问有没有重要日期 | ⭐⭐ 情感连接 |
| 番茄钟/专注模式 | Prism 屏幕显示倒计时 | ⭐⭐⭐ 硬件差异化 |

### 策略：Silent Onboarding（静默建档）
**永远不主动问用户任何个人信息。** 耐心听，仔细看，从用户的自然行为中提取一切。

```
用户行为                          → 推断
────────────────────────────────────────────
凌晨1点发消息                     → 夜猫子
"明天北京天气"                     → 在北京
"帮我看这段Python"                → 程序员/技术人员
连续3天23点还活跃                  → 晚睡型作息
发了张猫照                        → 养猫
"提醒我周三下午2点开会"            → 有固定会议习惯
"这个股票怎么看"                   → 关注投资
午饭时间没消息，下午1点突然活跃     → 午休到1点
"帮我翻译成英文"                   → 可能对外协作
周末消息明显少                      → 周末不太用/有其他安排
```

核心原则：
- ❌ 不问"你在哪个城市" — 等他提到或从天气查询推断
- ❌ 不问"你做什么工作" — 从他的问题类型和聊天内容推断
- ❌ 不问"你几点起床" — 从他的活跃时段推断
- ✅ 每一次对话、每一个请求都是一条线索
- ✅ 推断出来的信息静默写入用户画像，不跟用户确认
- ✅ 等信息自然积累到足够多时，用行动体现你懂他（而不是说"我发现你是夜猫子"）

### Prism 屏幕（冷启动期）
- 时间 + 天气 + 一句话（诗句轮换）
- 简单但不空——屏幕是"活着"的
- 第一次检测到用户 → 什么都不说，默默记住"主人到了"
- 用行动而非语言体现感知：人来了屏幕亮起来，走了暗下去——不需要任何解释

## Phase 1：观察期（Day 3-14，少量数据）

### 目标
开始展现"我在注意你"的能力，但**不急着给建议**。

### 数据积累
| 来源 | 能观察到的 |
|------|-----------|
| 对话记录 | 活跃时段、话题偏好、情绪词频 |
| 摄像头 presence | 在桌前的时间分布 |
| 录音转写 | 工作内容、社交圈、决策模式 |
| 主动提问 | 偏好、目标、习惯 |

### 开始做的事
1. **作息发现** — "你这几天都是 11 点以后才离开桌前，夜猫子型？"（不带判断，只是观察）
2. **话题追踪** — 用户提到的人名、项目、事件开始建档
3. **简单模式识别** — 周几最忙、什么时候心情好
4. **第一次"回忆"** — "你前天说想试那个XX，后来怎么样了？"（让用户惊喜："它记住了！"）

### 不做的事
- ❌ 不给生活建议（数据不够，给了也不准）
- ❌ 不做行为分析报告（太早了，像监控）
- ❌ 不预测行为（样本太少）

### Prism 屏幕进化
- 开始显示任务状态（用户手动设或 auto_status 推断）
- NOTE 区域：开始出现跟用户相关的内容（"今天有个会议"而不是诗句）

## Phase 2：理解期（Day 14-60，中等数据）

### 目标
从"观察"升级为"理解"，开始给出**有根据的轻量建议**。

### 新增能力
1. **行为模式报告**（每周一次，一条消息）
   - "这周你工作到 11 点有 3 天，比上周多了 1 天"
   - 不带评判，客观陈述 + 轻问一句
   
2. **第一批习惯建议**（每周 1-2 条）
   - 必须基于真实数据："你周二和周四到家最早，适合固定运动"
   - 不是"你应该多运动"这种废话
   
3. **想法回放**
   - "你两周前提到想做 XX，我找到一个相关的 YY，要不要看看？"
   
4. **社交图谱初步**
   - 知道谁是核心同事、午餐伙伴、决策时倾向跟谁商量
   
5. **情绪觉察**
   - 发现连续几天情绪低落或高压 → 轻触式关怀（不说"你不开心吧"，说"最近挺忙的，这个周末有个 XX 活动，要不要去？"）

### 建议追踪机制
```json
{
  "id": "sug_001",
  "date": "2026-03-17",
  "type": "exercise",
  "content": "周二 20:30 试试去亚C，你这天到家最早",
  "status": "pending",
  "evidence": "habit_data:tue_home_early",
  "followup_check": "2026-03-19",
  "outcome": null
}
```
- 给出建议后，标记 followup_check 日期
- 到了日期，从习惯数据中看用户那天有没有运动
- 采纳 → 同类建议权重 +1
- 忽略 → 权重 -0.5（不是惩罚，是学习偏好）

### Prism 屏幕进化
- DONE 区域开始有真实意义（一天做了什么一目了然）
- NOTE 开始出现主动提醒："今天有 XX 会议" / "记得回复 YY"
- 便签模式（18-20点）显示今天总结 + 明天计划

## Phase 3：参谋期（Day 60-180，丰富数据）

### 目标
成为用户的**决策参谋**——不只是提醒和建议，而是在关键时刻给出有价值的输入。

### 新增能力
1. **决策辅助**
   - 用户面临选择时，能快速拉出相关数据和历史决策
   - "你上次在 A 和 B 之间选了 A，结果是 XX，这次情况是..."
   
2. **行为预测**
   - 周二下午你大概率在开会 → 不打扰
   - 周五晚上你可能在找吃的 → 推荐餐厅
   - 月底你倾向于加班 → 提前问要不要调整计划
   
3. **深度习惯教练**
   - 不只是"你应该运动"，而是"你这个月运动了 8 次，比上月多 3 次，继续保持？"
   - 发现坏模式："连续 3 周你周三都加班到 12 点，这个会议能不能调一下？"
   
4. **信息过滤器**
   - 知道你关心什么 → 只推相关的新闻/趋势
   - 知道你的投资持仓 → 只在有重要变化时通知
   - 知道你的产品哲学 → 推送你会感兴趣的产品分析
   
5. **社交辅助**
   - "你已经两周没跟 XX 联系了，要不要约个饭？"
   - 记住重要日期：同事生日、纪念日

### Prism 屏幕进化
- 真正的"一眼看懂今天"：任务、提醒、天气、重要信息
- 离开时自动切到暗屏，回来时展示"你不在的时候发生了什么"
- 米家联动：人在 → 调灯、人走 → 灭灯

---

## Phase 4：知己期（Day 180+，深度数据）

### 目标
真正理解用户是谁、想要什么、需要什么——不需要用户说。

### 能力
1. **人生目标追踪**
   - 知道用户的长期目标，定期检查进展
   - "你年初说今年要 XX，现在进展到 YY 了，要不要聊聊？"
   
2. **第二大脑**
   - 记住用户说过的每个想法、每个决策
   - 在未来合适的时间调出来
   - "你去年 11 月想做的那个 RFID 项目，这周有个新的开源方案..."
   
3. **主动发现问题**
   - 从行为数据中发现异常：作息突然变化、社交锐减、情绪低落
   - 不是报警，是自然地关心
   
4. **跨域连接**
   - 把用户生活的不同方面连起来：工作压力 → 运动减少 → 睡眠变差 → 建议调整
   
5. **风格完全个性化**
   - 知道什么时候该正式、什么时候该随意
   - 知道什么建议用户会听、什么会反感
   - 沟通方式完全适应用户偏好
