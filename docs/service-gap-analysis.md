### ❌ 不适合做 / 暂时不可能

| 服务 | 原因 |
|------|------|
| **实时行程导航** | 需要实时录音流（我们是 T+1 拉取），且无导航 API |
| **GUI Agent 点外卖** | 需要手机端 Agent 执行能力，超出 Prism 范围 |
| **自动预约服务** | 需要对接各平台预约系统，不现实 |
| **轻应用生成** | 概念很好但对个人场景 ROI 低 |
| **育儿辅助** | 饭团没有小孩，不适用 |
| **演出自动选座购票** | 需要对接票务平台 API，且购票有身份验证 |

## 三、核心差距总结

### 我们有但没用好的

1. **录音数据利用率太低** — 每天 26 个场景、12h 录音数据只做了统计分析，没有"当天事件→当天服务"的实时闭环
2. **意图只存不动** — 287 条意图躺在 intents.json 里，只有过期提醒，没有主动触发行动
3. **情绪数据没用** — macro_frames 有 mood_or_tone，但从来没基于情绪状态做过服务
4. **会议数据是金矿** — 每天多个 meeting 场景，完全可以提取会议摘要+待办

### 我们缺的关键能力

1. **"当天录音→当天服务"快速闭环** — 目前 pipeline 是 T+1（23:40 跑），录音数据第二天才处理。show case 的核心是当天就给服务
2. **意图→行动自动触发器** — "提到想去福州"→自动查机票，"提到要办进京证"→自动设提醒
3. **结构化输出能力** — 会议纪要、待办清单、方案对比这些需要结构化模板

## 四、行动计划（按优先级）

### Phase 1：快速出效果（本周可做）

**目标：让录音数据每天产生可见价值**

1. **每日工作要点推送**
   - 从录音 macro_frames 提取 top 3-5 事件
   - LLM 生成简短总结（不超过 200 字）
   - 每晚 22:00 或次日 8:30 飞书推送
   - 改造 daily_pipeline.py 或新建 daily_brief.py

2. **会议摘要提取**
   - 过滤 scenes 中 activity=meeting 的场景
   - 提取 svo_bullets + key_quotes
   - 按会议分组生成简报
   - 集成到每日要点推送中

3. **情绪感知推送**
   - 检测 macro_frames 的 mood_or_tone
   - 如果下午/晚间出现 stressed/frustrated → 生成关怀消息
   - 融入 action.py 的 L1/L2 推送

### Phase 2：意图驱动行动（下周）

**目标：从"记录意图"升级到"执行意图"**

4. **意图自动分类+触发**
   - wish 类（想去某地/想买某物）→ 自动搜索+推方案
   - todo 类（要办某事）→ 创建跟踪任务+设提醒
   - idea 类（灵感/想法）→ 存入 idea-capture + 关联建议

5. **事件跟踪卡**
   - 检测重要事件（项目里程碑/个人事务）
   - 建立跟踪卡片，定期检查进展
   - 超期未完成 → 推送提醒

### Phase 3：深度服务（持续迭代）

6. **人际策略周报**
   - 基于录音社交数据生成"本周人际动态"
   - 识别关系升温/降温趋势
   - 给出 1-2 条具体建议

7. **明天预览**
   - 结合当天录音要点 + todo.md + 已知日程
   - 生成"明天你可能需要关注"的 brief
   - 晚间推送到 Prism 屏幕+飞书

8. **上下文感知推荐**
   - 录音提到某技术/产品 → 推送相关 AI 新闻
   - 录音提到某人 → 补充这个人最近的互动记录
   - 利用现有 master-search + rss-briefing

## 五、架构改造要点

### 录音处理从 T+1 改为 T+0（核心改造）

当前：录音 23:00 拉取 → 23:40 pipeline → 次日推送
目标：录音当天下班后即处理 → 当晚推送

方案：
- fetch_daily.py 改为 19:00 + 23:00 两次拉取
- 19:00 拉取当天截至下班的录音 → 生成当日 brief
- 23:00 拉取完整数据 → 跑 full pipeline

### action.py 增强

当前动作类型：exercise_reminder, social_suggestion, overtime_warning, intent_nudge, sleep_nudge, tomorrow_preview, weekend_plan

需新增：
- `daily_brief` — 每日工作要点
- `meeting_summary` — 会议摘要
- `emotion_care` — 情绪关怀
- `intent_action` — 意图自动触发
- `event_tracker` — 事件跟踪

### 推送通道利用

- 飞书消息（L2）：工作要点、会议摘要、意图提醒
- Prism 屏幕（L1）：情绪关怀、明天预览、运动提醒
- Prism 闪屏（L0.5）：实时事件通知
